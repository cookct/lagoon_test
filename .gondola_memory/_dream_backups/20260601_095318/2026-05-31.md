# Memory — 2026-05-31

## [gotcha] Array indices as message refs are broken
confidence: 0.95
Using array indices (msgIndex) to reference messages is fundamentally broken—indices become stale the moment any message is added or removed. This is systemic across the codebase with 91 uses of msgIndex.
---

## [decision] Fix: replace content in-place, don't splice
confidence: 0.85
The correct approach for message correction: keep the bad message object in its original position, reset its content to placeholder, let generateResponse push a new message, then copy the new content into the old message object and pop the duplicate. No index tracking needed.
---

## [gotcha] generateResponse always pushes to end
confidence: 0.9
The generateResponse function always pushes new assistant messages to the END of state.messages regardless of where the correction should go. This caused corrected responses to appear out of order at the bottom of the conversation.
---

## [gotcha] Abort path double-pop bug
confidence: 0.8
If user aborts generation and nothing streamed, generateResponse pops the new message. correctAndRegenerate then tries to pop again—removing the WRONG message. Fix: check if newMsg === badMsg before popping.
